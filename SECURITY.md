# Reporting a Vulnerability

If you believe you have discovered a security vulnerability in Vero, **please
report it responsibly**.

üö® **Do NOT report security vulnerabilities through public GitHub issues.**

---

# How to Report

Please send an email to:

**tiago@parceirosdenegocio.com**

With the subject line:

```
[Vero Security] Vulnerability Report
```

## Please include the following information:

- **Type of vulnerability**\
  (e.g., Cross-Site Scripting, Prototype Pollution, Denial of Service)

- **Full description**\
  Detailed explanation and clear steps to reproduce the issue

- **Impact**\
  How the vulnerability could be exploited and potential consequences

- **Proof of concept (if available)**\
  Code snippets, logs, or screenshots that help demonstrate the issue

Providing detailed information helps us resolve issues faster.

---

# What to Expect

After submitting your report:

- You will receive an acknowledgment within **48 hours**
- We will investigate the issue
- We may request additional information
- If confirmed, we will work on a fix as soon as possible
- A security release will be published
- You will be credited in the release notes (if you wish)

---

# Responsible Disclosure

We kindly ask that you:

- Allow reasonable time for investigation and patching
- Avoid public disclosure until a fix has been released
- Refrain from exploiting the vulnerability beyond proof-of-concept
